  • Generate the network graph
1. Download the code from the GitHub repo. Unzip.
2. Download your network data from the Facebook API. (apps.facebook.com/netvizz)
    -'personal network' -> start -> click 'gdf' and Save as...
3. Put this data inside the unzipped folder from GitHub.
4. On command line, go inside the InteractiveVis folder
5. Make script 'setup.sh' executable using: chmod u+x setup.sh
6. Run the GitHub script to install dependencies using: sudo ./setup.sh
    -This installs numpy, networkx, and pyparsing. It's important that the networkx installed in your Python is version 1.8.1.
7. Run the Python script to load data, detect communities, and layout nodes: python main.py
8. Load up a local server: python -m SimpleHTTPServer
9. On Firefox, go to the address: http://localhost:8000/network/?config=config_fb.json/

  • Mathematical analysis of the graph
1. Import the NetworkX library into Python: 
    import networkx as nx
2. Import the Counter object: 
    from collections import Counter
3. Import graph data into Python: G = nx.read_gml('facebook.gml')
4. This will allow you to export the nodes in a useable format, for example:
    cent = nx.degree_centrality(G)
    cent = Counter(cent)
    cent.most_common(10)
    -These functions combined will output the 10 most connected nodes in the graph. There are many other centrality measures. The documentation for NetworkX is here: http://networkx.lanl.gov/index.html

  • Discussion
After generating the network, we discussed what we could interpolate about one another from the graphs. Basically some in-depth, half-hour-to-an-hour of Facebook stalking.

We discussed algorithms to rank players by different measures of centrality (degree, closeness, betweenness) and see how the different measures lead to different people being pointed out in the network.

  • How do the graphs work?
Force directed graph drawing algorithms are used to generate the positions of the nodes on the graph.  In this case, the Fruchterman Reingold Algorithm has been used: nx.spring_layout(G). We used it because it's hard to install libraries that enable other layouts.  The Spring Layout was used but there are other layout functions that can be used (Reference material available on networkx.lanl.gov) e.g.: nx.graphviz_layout(G, prog='sfdp') 

Sometimes the Fruchterman Reingold Algorithm doesn't work as expected. If conducting this session among a group, you may be able to see examples of different networks that have been mapped with varying degrees of success.

The Clauset-Newman-Moore algorithm was used to automatically detect communities in the network.

It's hard to graph social networks because it's not possible to avoid crossing the edges. A graph cannot be drawn without edge crossings if it contains a subdivision of a complete graph on 5 nodes (K5) or that of a complete bipartite graph of size 3,3 (K3,3).

Terminology:
1. Centrality: computes the number of connections to other nodes on the graph.
2. Graph degree: number of connections a node has.
3. Distance = maximal number of connection points between two nodes.
4. Betweenness centrality
5. Closeness centrality

What needs to be worked on? What sort of help do you need?

  • Currently the algorithm just shows who's connected to who by an explicit link (friendship), but you could show other types of metadata like workplace, etc, which would add another informational layer to the graph.
  • Network graphs don't store sentiment! You may know someone and have many common connections with them, but have negative feelings towards that person and thus want to isolate them from the remainder of your network.  That's difficult to define without layers of additional information.
